# Skill: 創建個人作品集網站

## 觸發方式
當使用者輸入 `/創建網站` 時，執行此 Skill。

## 執行流程

### Step 1：收集資料
依序詢問使用者以下問題（一次問一個，等回答後再問下一個）：

1. 「你的名字是？」
2. 「一句話介紹自己（例如：設計師 / 攝影愛好者）」
3. 「你的 email 是？」
4. 「有幾個作品要展示？（1–6 個）」
5. 依作品數量，逐一詢問每個作品的：名稱、一句話描述、用什麼 emoji 代表
6. 「偏好深色還是淺色風格？」
7. 「GitHub 帳號名稱是？（用來推送程式碼）」
8. 「專案資料夾要建在哪裡？（直接輸入路徑，或按 Enter 使用桌面）」

---

### Step 2：建立資料夾與檔案

```bash
# 建立專案資料夾
mkdir -p {專案路徑}/my-portfolio
mkdir -p {專案路徑}/my-portfolio/images
cd {專案路徑}/my-portfolio
```

根據收集到的資料，生成 index.html，規則如下：

- 深色風格：背景 #0f0f0f，文字 #e8e8e8，強調色 #5b6af0
- 淺色風格：背景 #fafafa，文字 #1a1a2e，強調色 #5b6af0
- 作品卡片數量依使用者輸入動態生成
- 所有「你的名字」、「email」、「作品名稱」都替換成使用者提供的資料

生成的 index.html 結構：
1. nav — 名字 + 導覽連結
2. hero — 名字、一句話介紹、聯絡按鈕
3. about — 關於我（先留空，提示使用者之後填）
4. projects — 作品卡片（依數量自動生成）
5. contact — email 聯絡區塊
6. footer — 版權

---

### Step 3：Git 初始化與推送

依序執行：

```bash
git init
git add .
git commit -m "init: 建立個人作品集網站"
git branch -M main
git remote add origin https://github.com/{GitHub帳號}/my-portfolio.git
git push -u origin main
```

如果 push 失敗（例如 repo 不存在），告訴使用者：
「請先到 https://github.com/new 建立一個名為 my-portfolio 的 Public repo，建立後告訴我，我繼續推送。」

---

### Step 4：完成提示

推送成功後，顯示以下訊息：

```
✅ 完成！你的網站已建立並推上 GitHub。

下一步：部署到 Vercel（免費上線）
1. 前往 https://vercel.com
2. 用 GitHub 帳號登入
3. 點「Add New → Project」
4. 找到 my-portfolio，點「Import」
5. 直接按「Deploy」

部署完成後你會得到一個網址，例如：
https://my-portfolio-xxxxxxx.vercel.app

之後更新網站只需要執行：
  git add .
  git commit -m "更新說明"
  git push
```

---

## 注意事項

- 所有指令在使用者指定的資料夾內執行
- 如果資料夾已存在，詢問使用者是否覆蓋
- Git push 需要使用者已登入 GitHub，如果失敗引導他登入
- 圖片資料夾 images/ 建立好但留空，告訴使用者可以把圖片拖進去再 push
